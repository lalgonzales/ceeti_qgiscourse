Módulo 2: Carga y manejo de datos

1 Objetivos del Módulo

Al finalizar este módulo, los participantes podrán:

  • Importar datos geoespaciales en múltiples formatos (Shapefile, GeoJSON, KML, CSV).
  • Conectar QGIS a bases de datos como PostGIS y SQLite/SpatiaLite.
  • Comprender y gestionar los sistemas de referencia de coordenadas (CRS).
  • Visualizar rutas y redes de transporte en QGIS.

2 2.1 Importación de archivos espaciales

QGIS soporta una gran variedad de formatos de datos geoespaciales. A continuación, se describen los más comunes y cómo cargarlos.

2.1 Formatos vectoriales principales

Formato Extensión Descripción
Shapefile .shp Formato estándar de ESRI, muy difundido
GeoJSON .geojson Formato basado en JSON, ideal para web
GeoPackage .gpkg Formato moderno recomendado por OGC
KML/KMZ .kml, .kmz Formato de Google Earth/Maps
CSV .csv Tablas con columnas de coordenadas
GPX .gpx Formato GPS Exchange (tracks, waypoints)

2.2 Carga de archivos en QGIS

TipMétodos para cargar datos

Método 1 — Arrastrar y soltar: Arrastra el archivo desde el explorador de Windows/Mac directamente al canvas de QGIS.

Método 2 — Menú Capa: - Capa → Agregar capa → Agregar capa vectorial (Ctrl+Shift+V) - Capa → Agregar capa → Agregar capa ráster (Ctrl+Shift+R)

Método 3 — Panel del Explorador: Navega hasta el archivo en el panel Explorador y haz doble clic.

Método 4 — Drag & drop desde el explorador del sistema operativo.

2.3 Carga de CSV con coordenadas

Para cargar un archivo CSV con columnas de latitud y longitud:

  1. Capa → Agregar capa → Agregar capa de texto delimitado (Ctrl+Shift+T)
  2. Selecciona el archivo CSV
  3. En Geometría, elige:
    • Tipo de geometría: Punto
    • Campo X (longitud): columna con valores X
    • Campo Y (latitud): columna con valores Y
  4. Especifica el CRS del archivo
  5. Haz clic en Añadir

Ejemplo de CSV válido:

nombre,longitud,latitud
Hospital Central,-87.2068,14.0818
Mercado Los Dolores,-87.2041,14.0829
Parque Central,-87.2069,14.0839

3 2.2 Conexión a bases de datos

3.1 PostGIS (PostgreSQL + extensión espacial)

PostGIS es la extensión espacial para PostgreSQL más utilizada en entornos profesionales.

Pasos para conectar QGIS a PostGIS:

  1. Abre el Administrador de fuentes de datos (Ctrl+L)
  2. Selecciona la pestaña PostgreSQL
  3. Haz clic en Nuevo y completa:
    • Nombre de conexión: Mi_BaseDatos_PostGIS
    • Host: localhost (o IP del servidor)
    • Puerto: 5432
    • Base de datos: nombre de tu BD
    • Usuario y contraseña
  4. Haz clic en Probar conexión → Aceptar

3.2 SQLite / SpatiaLite

SpatiaLite es la versión espacial de SQLite, ideal para proyectos personales o de campo.

Capa → Agregar capa → Agregar capa SpatiaLite

3.3 GeoPackage

El formato GeoPackage (.gpkg) es un contenedor SQLite con capacidades espaciales:

Capa → Agregar capa → Agregar capa vectorial
→ Tipo de fuente: GeoPackage
NotaRecomendación de formato

Para proyectos nuevos, se recomienda usar GeoPackage en lugar de Shapefile, ya que: - Soporta nombres de campos de más de 10 caracteres - Almacena múltiples capas en un solo archivo - Es un estándar OGC abierto

4 2.3 Sistemas de referencia de coordenadas (CRS)

4.1 ¿Qué es un CRS?

Un Sistema de Referencia de Coordenadas (CRS) define cómo se relacionan las coordenadas de un mapa con las posiciones reales en la superficie terrestre.

4.2 Tipos de CRS

Tipo Descripción Ejemplo
Geográfico (GCS) Coordenadas en grados (lat/lon) WGS 84 (EPSG:4326)
Proyectado (PCS) Coordenadas en metros/pies sobre plano 2D UTM Zona 16N (EPSG:32616)
Local/Nacional Sistemas de cada país Marco Geodésico de Honduras

4.3 CRS para Honduras

Para trabajos en Honduras, se recomienda:

  • WGS 84 (EPSG:4326): para datos GPS y web
  • UTM Zona 16N (EPSG:32616): para análisis que requieren mediciones precisas
  • UTM Zona 17N (EPSG:32617): para el oriente del país

4.4 Gestión de CRS en QGIS

Ver el CRS de una capa: 1. Clic derecho sobre la capa → Propiedades 2. Pestaña Fuente → Sistema de referencia de coordenadas

Reproyectar una capa (cambiar el CRS): 1. Capa en el panel → clic derecho → Exportar → Guardar entidades como… 2. Cambia el CRS en el campo correspondiente 3. Guarda con el nuevo nombre

Transformación sobre la marcha (OTF): QGIS puede mostrar capas con diferentes CRS juntas activando la reproyección automática.

Advertencia¡Importante!

Si combinas capas con diferentes CRS, asegúrate de usar la reproyección en el análisis, no solo en la visualización. De lo contrario los cálculos de área y distancia serán incorrectos.

5 2.4 Visualización de rutas y redes de transporte

5.1 Carga de datos OpenStreetMap (OSM)

QGIS permite cargar datos de OpenStreetMap directamente:

Opción 1 — Plugin QuickOSM: 1. Instala el plugin: Complementos → Administrar e instalar complementos → QuickOSM 2. Ve a Vector → QuickOSM → QuickOSM 3. Consulta por ejemplo: clave highway, valor primary para carreteras principales 4. Define el área de interés y ejecuta

Opción 2 — Capas XYZ (fondo de mapa): 1. Panel Explorador → Teselas XYZ → clic derecho → Nueva conexión 2. URL: https://tile.openstreetmap.org/{z}/{x}/{y}.png 3. Nombre: OpenStreetMap

5.2 Datos de transporte en Honduras

Fuentes de datos espaciales para Honduras:

  • INE Honduras — Cartografía
  • OpenStreetMap Honduras
  • SINIT — Sistema Nacional de Información Territorial
  • HOT OSM — Tasking Manager Honduras

6 🛠️ Ejercicios Prácticos

6.1 Ejercicio 2.1: Carga de datos en múltiples formatos

  1. Descarga los datos de ejemplo:
    • Shapefile de municipios de Honduras
    • GeoJSON de carreteras principales
    • CSV con coordenadas de hospitales
  2. Carga cada archivo en QGIS usando métodos distintos.
  3. Verifica que se muestren correctamente en el canvas.

6.2 Ejercicio 2.2: Conexión a OpenStreetMap

  1. Instala el plugin QuickOSM.
  2. Descarga las carreteras principales de Tegucigalpa.
  3. Descarga los hospitales y centros de salud de la ciudad.
  4. Visualiza todas las capas juntas y ajusta la simbología básica.

6.3 Ejercicio 2.3: Trabajo con CRS

  1. Carga un shapefile con CRS geográfico (EPSG:4326).
  2. Calcula el área de un polígono en grados² — observa el resultado incorrecto.
  3. Reproyecta la capa a UTM Zona 16N (EPSG:32616).
  4. Calcula nuevamente el área y compara los resultados.

### Carga de datos OpenStreetMap (OSM)

QGIS permite cargar datos de OpenStreetMap directamente:

**Opción 1 — Plugin QuickOSM**:
1. Instala el plugin: **Complementos → Administrar e instalar complementos → QuickOSM**

3. Consulta por ejemplo: clave `highway`, valor `primary` para carreteras principales
4. Define el área de interés y ejecuta

**Opción 2 — Capas XYZ (fondo de mapa)**:

3. Nombre: `OpenStreetMap`

### Datos de transporte en Honduras
