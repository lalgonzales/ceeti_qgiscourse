Módulo 7: Estándares OGC, Servicios Web y Recolección de Datos en Campo

1 Objetivos del Módulo

Al finalizar este módulo, los participantes podrán:

  • Comprender los estándares OGC y su importancia en la interoperabilidad geoespacial.
  • Diferenciar entre servicios WMS y WFS y saber cuándo utilizar cada uno.
  • Conectar QGIS con el Geoportal del ICF de Honduras para consumir datos oficiales.
  • Implementar un flujo de trabajo completo de recolección de datos en campo usando QField y QFieldSync.

2 Estándares OGC e Interoperabilidad

En el mundo de los SIG, la capacidad de compartir datos entre diferentes plataformas (QGIS, ArcGIS, Google Earth, etc.) es fundamental. Esto es posible gracias al Open Geospatial Consortium (OGC).

2.1 ¿Qué es el OGC?

El OGC es una organización internacional no lucrativa que crea estándares abiertos para datos geográficos. Estos estándares aseguran que el software de diferentes proveedores pueda “hablar el mismo idioma”, permitiendo que un servidor de mapas en Linux sea leído por un cliente en Windows o Android sin problemas de compatibilidad.

TipInteroperabilidad

La interoperabilidad es la capacidad de un sistema para interactuar con otros sistemas. En SIG, esto significa que no dependemos de un único formato de archivo propietario, sino que utilizamos servicios web estandarizados para acceder a la información en tiempo real.

3 Servicios Web Geográficos: WMS y WFS

Los dos servicios más utilizados para distribuir información geográfica a través de la web son el WMS y el WFS.

3.1 WMS (Web Map Service)

Un servicio WMS entrega la información en forma de imágenes (generalmente PNG o JPEG). El cliente (QGIS) solicita una región del mapa y el servidor renderiza la imagen y la envía.

  • Ventaja: Carga muy rápida, no sobrecarga el cliente, ideal para mapas base.
  • Desventaja: No podemos acceder a los atributos originales ni editar la geometría; es solo una “foto” del mapa.

3.2 WFS (Web Feature Service)

Un servicio WFS entrega la información como vectores (coordenadas y atributos reales). El servidor envía los datos geográficos y el cliente (QGIS) los renderiza localmente.

  • Ventaja: Permite realizar análisis espaciales, filtrar atributos y, en algunos casos, editar los datos en el servidor.
  • Desventaja: Carga más lenta si hay muchos datos, ya que el cliente debe descargar toda la geometría.

3.3 Comparativa: WMS vs WFS

Característica WMS (Web Map Service) WFS (Web Feature Service)
Formato de entrega Imagen (Ráster) Vector (Geometría)
Carga de datos Rápida (renderizado servidor) Lenta (descarga de vectores)
Acceso a atributos Limitado (solo consulta) Total (tabla de atributos)
Análisis espacial ❌ No es posible ✅ Es posible
Edición ❌ Imposible ✅ Posible (si el servidor lo permite)
Uso principal Visualización y Mapas Base Análisis, Filtrado y Edición

4 Integración con el Geoportal del ICF

El Instituto de Conservación Forestal (ICF) de Honduras dispone de un Geoportal que permite el acceso a datos oficiales sobre el sector forestal y áreas protegidas mediante estándares OGC.

4.1 Acceso al Geoportal

El Geoportal del ICF es la fuente oficial para obtener capas de: - Áreas Protegidas de Honduras. - Red Vial Forestal. - Inventarios forestales y cobertura vegetal.

URL del Geoportal: https://geoportal.icf.gob.hn/geoportal/

4.2 Configuración de conexión WMS en QGIS

Para visualizar las capas del ICF como imágenes:

  1. Panel Explorador → WMS/WMTS \(\to\) clic derecho \(\to\) Nueva conexión
  2. Nombre: ICF - WMS
  3. URL: http://geoportal.icf.gob.hn:8081/geoserver/icf/wms
  4. Haz clic en Aceptar.
  5. Despliega la conexión en el panel Explorador y arrastra las capas deseadas (ej. icf:areas_protegidas) al mapa.

4.3 Configuración de conexión WFS en QGIS

Para descargar los vectores y analizar los datos del ICF:

  1. Panel Explorador → WFS \(\to\) clic derecho \(\to\) Nueva conexión
  2. Nombre: ICF - WFS
  3. URL: http://geoportal.icf.gob.hn:8081/geoserver/icf/wfs
  4. Haz clic en Aceptar.
  5. Despliega la conexión y arrastra la capa al mapa. Ahora puedes abrir la tabla de atributos y realizar consultas espaciales.
AdvertenciaRendimiento de WFS

Cuando utilices WFS con capas muy extensas (como la red vial nacional), evita cargar toda la capa. Utiliza el filtro de la conexión o haz zoom en un área pequeña antes de solicitar los datos para evitar que QGIS se bloquee.

5 Recolección de datos en campo con QField y QFieldSync

La recolección de datos en campo es una fase crítica de cualquier proyecto SIG. QField es una aplicación móvil diseñada para llevar la potencia de QGIS al dispositivo Android o iOS.

5.1 ¿Qué es QField?

QField no es solo un visor de mapas; es una herramienta de captura de datos que permite: - Visualizar mapas base y capas oficiales en el terreno (offline). - Crear nuevas entidades (puntos, líneas, polígonos) con el GPS del móvil. - Completar formularios de atributos personalizados creados en QGIS.

5.2 QFieldSync: El puente entre escritorio y campo

Para que un proyecto de QGIS funcione en QField, debe ser “empaquetado” y configurado. Para esto utilizamos el plugin QFieldSync.

Flujo de Trabajo Profesional:

1. Preparación (QGIS Desktop) - Diseñar la simbología y etiquetas. - Crear formularios de entrada de datos (listas desplegables, validaciones). - Configurar el plugin QFieldSync para definir qué capas son editables y cuáles son solo lectura.

2. Transferencia - Exportar el proyecto mediante QFieldSync. - Transferir la carpeta al dispositivo móvil (vía USB, QFieldCloud o almacenamiento compartido).

3. Captura (QField Mobile) - Abrir el proyecto en el dispositivo. - Recolectar datos en campo usando el GPS. - Guardar los cambios localmente en el dispositivo.

4. Sincronización (Retorno) - Transferir la carpeta del dispositivo de vuelta al computador. - Usar QFieldSync para Sincronizar los datos capturados con la base de datos original de QGIS.

5.3 Configuración de capas en QFieldSync

Al configurar el proyecto, debemos asignar un “Action” a cada capa:

Acción Descripción Uso típico
No Action La capa no se exporta al móvil Capas temporales o muy pesadas
Offline Editing La capa es editable en campo Capas de puntos de muestreo, incidencias
ReadOnly La capa se visualiza pero no se edita Mapas base, límites administrativos, ríos

6 🛠️ Ejercicio Práctico

6.1 Ejercicio 7.1: Consumo de servicios oficiales del ICF

  1. Configura una conexión WMS al Geoportal del ICF.
  2. Carga la capa de Áreas Protegidas y ajusta su transparencia para verla sobre un mapa satelital.
  3. Configura una conexión WFS al Geoportal del ICF.
  4. Carga la misma capa de Áreas Protegidas vía WFS.
  5. Abre la tabla de atributos y filtra las áreas protegidas que tengan una superficie mayor a un valor específico.

6.2 Ejercicio 7.2: Preparación de proyecto para QField

  1. Crea un nuevo proyecto de QGIS y añade una capa de puntos llamada Muestreo_Campo (GeoPackage).
  2. Configura el formulario de la capa con los siguientes campos:
    • ID (Autoincrementado)
    • Fecha (Fecha)
    • Observacion (Texto)
    • Estado (Valor relación / Lista: Bueno, Regular, Malo)
  3. Instala el plugin QFieldSync.
  4. Configura la capa Muestreo_Campo como Offline Editing y el mapa base como ReadOnly.
  5. Empaqueta el proyecto y simula la exportación hacia un dispositivo móvil.
